# ExecPlan Process

Use an ExecPlan for work that spans multiple steps, files, or validation loops.

## Required Sections

- `Goal`
- `Scope`
- `Plan`
- `Progress`
- `Surprises & Discoveries`
- `Decision Log`
- `Validation`
- `Outcomes & Retrospective`

## Rules

- Keep the plan living and current while the work is in flight.
- Record changes in assumptions, regressions, or new findings as they appear.
- Prefer small, direct steps that can be validated independently.
- Close the plan with the final validation outcome and any follow-up work.

## When To Use

- Multi-file refactors
- Cross-module bug fixes
- Version or dependency upgrade work that needs staged verification
- Any task where the next step depends on the result of earlier steps
